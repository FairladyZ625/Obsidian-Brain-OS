# 摩擦归桶指南

分类摩擦信号时，使用这些标准桶。

## 标准桶

### 1. Prompt 歧义
执行说明允许多种解读。
- 典型案例：只说“写计划”但不指定写到哪里
- 通常回写：**Prompt**

### 2. Reference 遗漏
可复用规则存在但被跳过或遗忘。
- 典型案例：knowledge-plan-governance 存在但写入前未读取
- 通常回写：**AGENTS** 或 **Reference**

### 3. AGENTS 可见性缺口
规则应在行动前可见，但被埋在深层文档或缺失于 AGENTS。
- 典例：“先查 reference”只在深文档里，不在 AGENTS 里
- 通常回写：**AGENTS**

### 4. 工作流盲区
自动化或扫描逻辑持续抓不到真正的问题。
- 典型案例：daily sync 只扫文件但看不到系统级升级
- 通常回写：**Workflow**（cron prompt / 扫描脚本）

### 5. 命名 / 结构混淆
名称或目录结构误导 agent 对落点的判断。
- 典例：workspace 看起来像正式计划的合法落点
- 通常回写：**Reference** 或 **Docs/Onboarding**

### 6. 边界 / 隐私问题
私有内容泄露到公开输出，或反之。
- 典例：内部 agent 名出现在 OSS 文档中
- 通常回写：**Reference** 或 **Workflow**（PII 扫描器 / 配置泄露扫描器）

### 7. Onboarding 缺口
新能力存在但安装 / 入门文档从未提及。
- 典例：治理栈已加但 INSTALL_FOR_AGENTS 未更新
- 通常回写：**Docs/Onboarding**

### 8. 结构漂移
系统已经演进但旧文档 / prompt / 配置未同步。
- 典例：component-guide 还停在 v0.5 但最新版已是 v1.1.0
- 通常回写：**Docs**（多文件）

## 使用方法

处理摩擦信号时：
1. 逐条读取信号
2. 归入一个标准桶（如都不匹配可新建）
3. 统计每桶数量
4. 有 2+ 信号的桶成为治理回写候选
5. 只有 1 个信号的桶保持“观察”

## 何时新建桶
如果某个问题在 10+ 条信号后仍无法归入现有桶，考虑是否需要新建。

不要为一次性问题新建桶。
