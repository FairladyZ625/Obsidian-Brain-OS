# 04 — 迭代指南

> 这套系统不是一次性配好的。它需要你持续迭代，让它越来越适合你。

---

## 为什么需要迭代

每个人的习惯不同：
- 有人喜欢早上看摘要，有人喜欢晚上
- 有人一周处理 50 篇文章，有人一周 5 篇
- 有人需要详细的每日驾驶舱，有人只需要 3 行提醒

Brain OS 提供的是**经过验证的默认配置**。但默认配置不一定适合你。

**迭代的目标：让系统越来越贴合你的实际工作方式。**

---

## 迭代什么

### 1. Prompt 迭代

Prompt 是 Agent 行为的"基因"。微调 Prompt 比什么都有效。

**高频迭代点**：

| Prompt | 常见调整 | 效果 |
|--------|---------|------|
| `personal-ops-morning-brief` | 驾驶舱的结构、优先级判断逻辑 | 直接影响每日体验 |
| `article-integration` | 知识卡的详细程度、何时创建新卡 | 影响知识库质量 |
| `conversation-mining` | 什么算"有价值的洞察" | 影响对话挖掘的效果 |

**怎么改**：
1. 观察 Agent 的输出
2. 找到不满意的地方
3. 修改对应 Prompt 文件
4. 第二天看效果
5. 重复

**原则**：每次只改一个地方，这样才能知道改了什么、效果是什么。

### 2. Skill 迭代

Skill 是 Agent 的"专业技能"。比 Prompt 更高级。

**什么时候加新 Skill**：
- 你发现自己在某个场景反复给 Agent 相同的指令
- Agent 在某个场景总是做不好
- 你有一个标准化的工作流想让 Agent 遵循

**怎么加**：
1. 参考 `skills/brain-os-installer/SKILL.md` 的格式
2. 写清楚：触发条件、执行步骤、输出格式、禁止事项
3. 放入 `skills/` 目录
4. 在 `openclaw.json` 的 `agents.main.skills` 中注册

### 3. 频道迭代

**什么时候拆频道**：
- 一个频道的 Agent 经常被不相关的话题干扰
- 你发现自己经常说"这个不是这个频道该讨论的"

**什么时候合并频道**：
- 某个频道几乎没消息
- 两个频道的职责边界模糊

### 4. Cron Job 迭代

**什么时候调整时间**：
- 02:00 Pipeline 总是超时 → 改到 03:00，让文章先积累
- 07:00 驾驶舱太早 → 改到 08:00
- 你习惯晚睡 → Pipeline 改到 04:00-06:00

**什么时候加新 Cron Job**：
- 你发现某个事情每天都在手动做 → 自动化它
- 例如：每天检查邮件、每周写周报、每月做复盘

### 5. 知识库结构迭代

**什么时候调整目录**：
- 某个目录太深了，你从来不去
- 某个目录太浅了，内容太多找不到
- 你发现一种新的知识类型没有合适的存放位置

**原则**：目录结构不是神圣的。改了之后更新模板和 Skill 里的路径引用。

## 迭代节奏

### 每天

- 观察 Agent 输出质量
- 记录不满意的地方（可以写在 `memory/` 里）

### 每周

- 调整 1-2 个 Prompt
- 看一次 knowledge-lint 报告
- 清理 `00-INBOX/` 中已完成的事项

### 每月

- 审视频道分工是否合理
- 检查 Cron Job 执行成功率
- 看一次 `MEMORY.md`，更新长期记忆
- 考虑是否需要新 Skill

### 每季度

- 审视整体系统架构
- 考虑是否需要新的频道或工具
- 回顾知识库增长情况
- 分享你的经验（如果愿意的话）

## 常见的迭代方向

### 方向 1：更精准的驾驶舱

初期驾驶舱可能太长或太短。迭代方向：
- 减少"低价值"提示（如已完成的旧事项）
- 增加"高价值"提示（如即将到期的承诺）
- 调整优先级判断逻辑

### 方向 2：更好的知识提取

初期知识卡可能太泛或太窄。迭代方向：
- 调整 article-integration 的提取粒度
- 增加领域特定的提取规则
- 优化 pattern detection 的灵敏度

### 方向 3：更多自动化

初期你可能很多事手动做。迭代方向：
- 把手动操作变成 Cron Job
- 把重复指令变成 Skill
- 让 Agent 主动提醒而不是被动等待

### 方向 4：多语言支持

如果你主要用中文：
- 知识卡可以默认中文
- 驾驶舱可以中文
- Skill 指令可以中文
- 但 frontmatter 字段建议保留英文（兼容性）

## 迭代的反模式

### ❌ 一次改太多

一次改 5 个 Prompt + 3 个 Skill + 2 个 Cron Job。改完之后出问题，不知道是哪个改坏了。

**正确做法**：每次只改一个，观察效果，再改下一个。

### ❌ 不观察就改

感觉"好像不太好"就改。没有具体的不满意点。

**正确做法**：记录具体问题（"今天的驾驶舱有 3 个已完成事项不该出现"），针对性修改。

### ❌ 改完不验证

改了 Prompt 就不管了。第二天发现更差了也不知道。

**正确做法**：改完之后，第二天对比前后输出，确认效果。

### ❌ 追求完美

永远在改配置，从来不在实际工作中使用系统。

**正确做法**：80% 好就用起来。剩下的 20% 在使用中慢慢优化。

## 我们提供的 vs 你需要自己做的

### 我们提供的（开箱即用）

- ✅ 目录结构模板
- ✅ 文档模板（article note / daily brief / weekly plan / project brief）
- ✅ Nightly Pipeline（3 阶段自动知识处理）
- ✅ 自动化脚本（knowledge-lint / init-nightly-digest / export-conversations）
- ✅ 核心 Skills（6 个）
- ✅ 推荐 Skills（16 个）
- ✅ Cron 配置示例
- ✅ 完整文档和教程

### 你需要自己做的

- 🔧 配置你的 OpenClaw Agent（JSON）
- 🔧 设计你的频道分工
- 🔧 调整 Prompt 适配你的习惯
- 🔧 持续迭代和优化
- 🔧 坚持使用（至少一个月）

**Brain OS 是一副骨架。你需要在上面长出自己的血肉。**
